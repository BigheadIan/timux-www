Step 1: 检查系统环境

先确认你的 Mac 满足要求。打开 终端（按 ⌘+空格，输入「终端」回车），然后运行：

检查 macOS 版本和芯片：
sw_vers && uname -m
检查磁盘空间（需要 ≥500MB）：
df -h / | tail -1 | awk '{print "可用空间: "$4}'

✅ 看到 arm64 = Apple Silicon（M 系列），x86_64 = Intel

常见问题

  • 找不到终端？ 按 ⌘+空格 打开 Spotlight，输入 Terminal 或「终端」
  • 不是 arm64？ 你的 Mac 是 Intel 芯片，目前脚本也支持，可以继续
  • 空间不足？ 清理一些文件后再试
📦
Step 2: 安装 Homebrew + Node.js

Homebrew 是 Mac 的软件包管理器，Node.js 是运行 AI 管家的基础。

① 安装 Homebrew（如果已有会提示，不用重装）：
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

⚠️ 可能需要输入 Mac 开机密码（输入时不显示，正常现象）

② Apple Silicon Mac 需要多运行这一步：
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile && eval "$(/opt/homebrew/bin/brew shellenv)"
③ 安装 Node.js 22：
brew install node@22 && brew link node@22 --force --overwrite
④ 验证安装：
node --version && npm --version

应该看到 v22.x.x 和 10.x.x

常见问题

  • Homebrew 安装卡住？ 可能是网络问题，等几分钟或换个网络重试
  • 已经装了 Node.js？ 运行 node --version，如果是 v22 以上就不用重装
  • brew 命令找不到？ 关闭终端重新打开，或运行第②步的命令
  • 需要密码？ 输入你开机时用的密码（输入时看不到字符是正常的）
🦞
Step 3: 安装 OpenClaw（AI 管家引擎）

一行命令安装 AI 管家核心，约 1-2 分钟。

安装 OpenClaw：
npm install -g openclaw@latest
验证安装：
openclaw --version

应该看到版本号，比如 0.x.x

常见问题

  • npm 命令找不到？ 先完成 Step 2，确保 Node.js 安装成功
  • 权限错误？ 试试 sudo npm install -g openclaw@latest
  • 下载很慢？ 等几分钟，npm 有时候比较慢
🧠
Step 4: 配置 AI 大脑 + 人格

设置 API Key 并运行 OpenClaw 初始化向导。

📖 如何获取 API Key？

  1. 打开 platform.deepseek.com，注册/登录
  2. 点击左侧 「API Keys」
  3. 点击 「Create new API key」
  4. 复制 Key（格式：sk-xxxxxxxx）

💡 新注册送 500 万 tokens（约 ¥10），足够用好几天

  1. 打开 dashscope.console.aliyun.com，用支付宝/淘宝登录
  2. 先 「开通 DashScope」（免费）
  3. 右上角头像 → 「API-KEY 管理」 → 「创建新的 API-KEY」
  4. 复制 Key

💡 Qwen Turbo 有免费额度

  1. 打开 open.bigmodel.cn，注册/登录
  2. 进入 「开发工作台」 → 「API Keys」 → 创建
  3. 复制 Key

💡 GLM-4-Flash 完全免费！

拿到 API Key 后，运行 OpenClaw 初始化向导：
openclaw onboard --install-daemon

向导会引导你选择模型提供商、输入 API Key、设置密码。按照提示操作即可（约 2 分钟）。

⚠️ 如果提示输入 Mac 密码，这是为了安装开机自启动服务（正常现象）

常见问题

  • 选哪个模型？ 国内推荐 DeepSeek（便宜快速），海外推荐 Claude 或 GPT-4
  • API Key 输错了？ 之后可以改配置：openclaw config edit
  • 向导卡住？ 按 Ctrl+C 取消，重新运行
🚀
Step 5: 验证安装

最后一步！点击下面的按钮，自动检测 AI 管家是否成功运行。

或者手动检查：
openclaw gateway status

检测失败？

  • 服务没启动？ 运行 openclaw gateway start
  • 端口被占用？ 运行 lsof -i :18789 看看谁在用
  • 查看日志： openclaw gateway logs
  • 重启试试： openclaw gateway restart
