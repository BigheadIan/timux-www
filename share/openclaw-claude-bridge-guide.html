一、架構總覽

OpenClaw agent Bridge Claude CLI (Discord/Telegram) │ │ │ │ POST /v1/chat/ │ │ │ completions │ 轉換訊息格式 │ ├───────────────────────▶│ 注入工具協議 │ │ │ 映射思考等級 │ │ │ │ │ │ stdin ──▶ claude --print │ │ │ stdout ◀── stream-json │ │ │ │ │ SSE stream │ 解析回應： │ ◀───────────────────────│ ├─ 文字 → 乾淨輸出 │ │ │ └─ <tool_call> → OpenAI │ │ │ │ │ tool_calls? │ │ │ 有 → OC 執行工具 │ │ │ 無 → 答案回傳用戶 │ │
核心原則：Bridge 從不執行工具。它只負責翻譯。OpenClaw 擁有工具循環。

二、前置條件

  • Claude Code CLI 已安裝並登入 Max 訂閱帳號
  • Node.js >= 18
  • OpenClaw >= 2026.1（Gateway 運行於 port 18789）
# 驗證
claude --version
claude auth status   # 確認已登入 Max 帳號

三、Bridge 安裝

# 1. Clone
git clone https://github.com/shinglokto/openclaw-claude-bridge.git
cd openclaw-claude-bridge
npm install   # 會自動 build Dashboard

# 2. 設定 .env
cp .env.example .env
# 編輯 .env，至少設定 DASHBOARD_PASS

# 3. 啟動測試
npm start
curl http://localhost:3456/health   # 應回傳 {"status":"ok"}

端口說明

端口綁定用途
3456localhostOpenAI 相容 API（僅本機）
34580.0.0.0Dashboard（區域網絡可存取）

重要環境變數

變數預設說明
DASHBOARD_PASS—Dashboard 密碼（用戶名 admin）
IDLE_TIMEOUT_MS120000Claude CLI 無輸出超時（ms）
MAX_PER_CHANNEL2每 channel 最大同時請求數
MAX_GLOBAL20全局最大同時請求數
OPUS_MODELopusOpus 的 CLI 模型別名
SONNET_MODELsonnetSonnet 的 CLI 模型別名

四、macOS 開機自啟（launchd）

# 一鍵安裝（自動偵測路徑、讀取 .env）
./service/install-mac.sh

手動管理

# 重啟（重載 plist 設定）
launchctl bootout gui/$(id -u)/com.openclaw.claude-bridge
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.openclaw.claude-bridge.plist

# 日誌
tail -f ~/openclaw-claude-bridge/bridge.log
⚠️ 重要：.env 改動後要重跑 ./service/install-mac.sh，因為 launchd 在啟動時就載入環境變數。

Linux（systemd）

cp service/openclaw-claude-bridge.service ~/.config/systemd/user/
systemctl --user daemon-reload
systemctl --user enable --now openclaw-claude-bridge
loginctl enable-linger $USER  # 未登入時亦可開機啟動

五、OpenClaw 設定

在 ~/.openclaw/openclaw.json 新增 provider：

{
  "models": {
    "providers": {
      "claude-bridge": {
        "baseUrl": "http://localhost:3456/v1",
        "apiKey": "not-needed",
        "api": "openai-completions",
        "models": [
          {
            "id": "claude-opus-latest",
            "name": "Claude Opus",
            "contextWindow": 1000000,
            "maxTokens": 128000,
            "reasoning": true
          },
          {
            "id": "claude-sonnet-latest",
            "name": "Claude Sonnet",
            "contextWindow": 1000000,
            "maxTokens": 64000,
            "reasoning": true
          }
        ]
      }
    }
  }
}

然後將 agent 的模型指定為 claude-bridge/claude-opus-latest 或 claude-bridge/claude-sonnet-latest。apiKey 可以是任何非空字串 — Bridge 不會驗證。

六、Context 優化（三層防護）

第一層：Agent 自檢

Heartbeat 時自動檢查 workspace 檔案大小，超過預算立即壓縮：

檔案預算
WORKLOG.md< 2K chars
MEMORY.md< 3K chars
CONTEXT.md< 5K chars
for f in WORKLOG.md MEMORY.md CONTEXT.md; do
  SIZE=$(wc -c < "$f" 2>/dev/null || echo 0)
  echo "$f: ${SIZE} bytes"
done

第二層：Bridge 告警

Prompt 超過 50K tokens → Bridge log WARNING，提醒需要壓縮。

第三層：Session 壓縮

Session 接近 context 上限時觸發 memoryFlush + compact refresh，自動持久化記憶並重新整理上下文。

七、清除污染 Session（400 錯誤處理）

Session 被污染（400 Bad Request）時，需清除三層：

# 1. Bridge state
rm ~/openclaw-claude-bridge/state.json

# 2. Claude CLI sessions
claude sessions list
# 手動刪除有問題的 session，或清除全部：
# claude sessions clear

# 3. OpenClaw sessions
# 在 OC 內執行 /new 建立新 session

九、注意事項
